Install Ubuntu in VirtualBox

SA101 — System Administration and Maintenance · Lesson 1

This lesson walks you through the exact SA101 procedure for installing Ubuntu Desktop in Oracle VirtualBox. Since you have 24 GB RAM, we'll allocate 8192 MB and 4 processors — well inside the safe zone.

Purpose: You're doing this first as a student, then you'll teach it. Pay attention to where students might get confused.

Before you start

You already have the files downloaded:

  • VirtualBox-7.2.12-174389-Win.exe
  • ubuntu-26.04-desktop-amd64.iso
Safety first. This installs Ubuntu inside a VM only. Do NOT modify, shrink, or format your real Windows drive. The Ubuntu installer's "Erase disk" option is safe only because it sees the virtual disk, not your real one.

Step 1 — Check virtualization is enabled

  1. Press Ctrl + Shift + Esc → Task Manager
  2. Click Performance → CPU
  3. Check "Virtualization" shows Enabled
If it's disabled, do not change BIOS settings yourself. Inform your instructor. BIOS/UEFI varies by laptop brand.

Step 2 — Install VirtualBox

  1. Double-click VirtualBox-7.2.12-174389-Win.exe
  2. Click Yes if Windows asks permission
  3. Next → keep defaults → Next
  4. When warned about network interruption, save any online work, then Install
  5. Allow driver installation
  6. Keep "Start Oracle VirtualBox" checked → Finish

Step 3 — Create the VM

  1. In VirtualBox Manager, click New
  2. Fill in the first screen:
    FieldValue
    VM NameSA101-Ubuntu-{yourname} (e.g. SA101-Ubuntu-Rex)
    VM FolderDefault
    ISO ImageClick the dropdown → Other → select ubuntu-26.04-desktop-amd64.iso
    OSLinux
    OS DistributionUbuntu
    OS VersionUbuntu (64-bit)
  3. Uncheck "Proceed with Unattended Installation" — it is checked by default.
    If you leave this checked, VirtualBox will create a default "vboxuser" account and skip the Ubuntu installer. Uncheck it so you (and your students) see and perform every installation step manually.
    After unchecking, the blue info box at the bottom should disappear, and you'll go straight to the hardware screen next.
  4. Click Next

Step 4 — Specify virtual hardware (24 GB RAM host)

VirtualBox 7.2 shows all hardware settings on one screen when unattended installation is unchecked:

ResourceValue
Base Memory8192 MB
Number of CPUs4
Disk Size40.00 GB
Use EFIUnchecked (default)
Students with 8 GB hosts should use 4096 MB / 2 CPUs. With 16 GB: 6144 MB / 2-4 CPUs. Keep sliders in the green zone.

Click Next → review summary → Finish.

Step 5 — Quick settings check

  1. Select the VM → Settings
  2. Check these:
    • System: RAM and CPU correct, optical drive in boot order
    • Display: Graphics Controller = VMSVGA, Video Memory = 128 MB
    • Storage: Ubuntu ISO attached to optical drive
    • Network: Adapter 1 enabled, Attached to: NAT
  3. Click OK

Step 6 — Install Ubuntu

  1. Select the VM → Start
  2. You'll see the GRUB boot menu. Select Try or Install Ubuntu and press Enter.
    3. Display glitch? If you get a black screen or garbled graphics, restart and choose Ubuntu (safe graphics) instead. It uses basic display drivers.
  3. Wait for the Ubuntu installer to load (this takes a moment)
  4. Work through the installer:
    ScreenChoice
    LanguageEnglish
    AccessibilityDefault
    KeyboardEnglish (US) or your layout
    NetworkConnect (NAT provides internet automatically)
    TypeInstall Ubuntu (not Try)
    Installation typeInteractive Installation
    ApplicationsDefault Selection
    Third-partyCheck both options when internet is available (see below)
    Third-party options explained:
    ☑ Graphics & Wi-Fi drivers — installs NVIDIA and similar proprietary drivers. In a VM this has little effect, but it's harmless.
    ☑ Additional media formats — installs MP3, MP4, MOV codecs. Useful if you play media files.

Step 7 — Disk & account

  1. When asked: "Erase disk and install Ubuntu"
    This is safe ONLY because you're inside a VM. Never choose this on a physical install unless you're sure which disk you're erasing.
  2. On the Encryption and file system screen, select No encryption.
    Encryption is unnecessary for a classroom VM and adds a passphrase prompt on every boot. Leave it off.
  3. Set your account:
    FieldExample
    Your NameRex
    Computer Namesa101-ubuntu-surname (lowercase, hyphens, no spaces)
    UsernameYour lowercase username
    PasswordStrong password — remember it
    Login optionRequire my password to log in
  4. Select time zone (Asia/Manila for PH) → Install
  5. Wait for installation to complete → Restart Now
Installer freeze on "Copying files"? The VM likely has too little RAM or CPU. Power off, go to Settings → System → Processor, and increase to at least 4096 MB RAM / 2 CPUs. Restart — it will resume the install.
"Please remove the installation medium, then press Enter" — this is normal.
In the VirtualBox menu: Devices → Optical Drives → Remove disk from virtual drive, then press Enter.
If the option is greyed out, just press Enter — VirtualBox often auto-ejects it.
If Ubuntu restarts into the installer instead of the desktop, shut down, go to Settings → Storage, remove the ISO, and start again.

Step 8 — First login & updates

  1. Log in with the account you created
  2. Open Terminal: Ctrl + Alt + T
  3. Run these commands one at a time: 
    sudo apt update
sudo apt upgrade -y
sudo snap refresh
  4. Reboot: 
    sudo reboot

Step 9 — Guest Additions

Guest Additions improves screen resolution, mouse integration, shared clipboard, and drag-and-drop. Two methods to install:

Method 1 — Graphical (try this first)

  1. Start Ubuntu and log in
  2. In VirtualBox menu: Devices → Insert Guest Additions CD Image
  3. Ubuntu may show a prompt — click Run
  4. Enter your password → wait for installation to finish
  5. Restart Ubuntu

If the graphical installer fails with bzip2 not found, run this in a terminal and retry:

sudo apt install bzip2 tar -y

Method 2 — Terminal fallback

If the graphical method doesn't work at all:

  1. Open Terminal
  2. Install dependencies: 
    sudo apt install build-essential dkms linux-headers-$(uname -r) -y
  3. Insert the CD image: Devices → Insert Guest Additions CD Image
  4. Open the Files app, open the mounted VBox_GAs disk
  5. Right-click inside the folder → Open in Terminal
  6. Run: 
    sudo sh ./VBoxLinuxAdditions.run
  7. Restart: 
    sudo reboot

After restart — enable the features

  1. View → Auto-resize Guest Display (makes the VM resize with the window)
  2. Devices → Shared Clipboard → Bidirectional (copy/paste between host and guest)
  3. Devices → Drag and Drop → Bidirectional (drag files between host and guest)

Step 10 — Verify

whoami
hostnamectl
ip a
lsblk
df -h
free -h
ping -c 4 8.8.8.8
ping -c 4 ubuntu.com

Step 11 — Snapshot

  1. Shut down Ubuntu properly: Power menu → Power Off
  2. In VirtualBox Manager: select VM → Snapshots → Take
  3. Name: Clean-Install-Updated

Checklist

  • Virtualization is enabled
  • VirtualBox installed
  • VM created with correct name and ISO
  • "Proceed with Unattended Installation" unchecked
  • 8192 MB RAM, 4 CPUs, 40 GB disk allocated
  • Ubuntu installed on virtual disk
  • Account and computer name set correctly
  • Ubuntu updates complete
  • Guest Additions installed
  • Verification commands all pass
  • Clean snapshot taken
