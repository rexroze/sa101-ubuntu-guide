Installing Ubuntu Desktop in Oracle VirtualBox

SA101 — System Administration and Maintenance · Step-by-Step Guide

IMPORTANT SAFETY RULES
1. This activity installs Ubuntu INSIDE A VIRTUAL MACHINE only.
2. Do NOT install Ubuntu directly on your laptop.
3. Do NOT create a dual-boot setup.
4. Do NOT modify, delete, shrink, or format your real Windows drive.
5. When Ubuntu says "Erase disk," it must refer only to the virtual disk inside VirtualBox.
6. Back up important files before starting.
7. Do not show your password in screenshots or reports.

1Check your computer

Your Windows PC needs these minimum specs:

RequirementMinimumRecommended
OSWindows 10 or 11, 64-bit
RAM8 GB16 GB+
Free storage40–50 GB
InternetStable connection
VirtualizationEnabled in hardware

Check if virtualization is enabled:

  1. Press Ctrl + Shift + Esc → Task Manager
  2. Click Performance → CPU
  3. Look for "Virtualization" — it should say Enabled
If it says Disabled, do NOT change BIOS settings yourself. Tell your instructor. BIOS/UEFI varies by laptop brand.

2Download Ubuntu Desktop

  1. Go to https://ubuntu.com/download/desktop
  2. Find the latest Ubuntu Desktop LTS release
  3. Select the Intel or AMD 64-bit download
  4. Wait for the download to finish
  5. Confirm the file ends in .iso (e.g. ubuntu-26.04-desktop-amd64.iso)
Do not extract or open the ISO file. Just remember where you saved it.

3Download and install Oracle VirtualBox

  1. Go to https://www.virtualbox.org/wiki/Downloads
  2. Under "VirtualBox Platform Packages," click Windows hosts
  3. Run the downloaded installer (ends in -Win.exe)
  4. If Windows asks "Do you want to allow this app to make changes?" → click Yes
  5. Next → keep defaults → Next
  6. VirtualBox will warn that the network will temporarily disconnect. Save any online work, then continue
  7. Click Install → allow driver installation
  8. Keep "Start Oracle VirtualBox" checked → Finish
The VirtualBox Extension Pack is not required for this activity.

4Create the virtual machine

  1. Open Oracle VirtualBox → click New
  2. Fill in the first screen:
    FieldValue
    VM NameSA101-Ubuntu-{yourname}
    VM FolderDefault
    ISO ImageClick dropdown → Other → select your .iso file
    OSLinux
    OS DistributionUbuntu
    OS VersionUbuntu (64-bit)
Critical step: Uncheck "Proceed with Unattended Installation" — it is checked by default. If you leave it checked, VirtualBox creates a default "vboxuser" account and skips the Ubuntu installer. You won't see the installation steps.
  1. Click Next

5Allocate RAM, CPUs, and disk

VirtualBox 7.2 shows all hardware on one screen. Set these values:

Resource8 GB RAM host16 GB RAM host24 GB+ RAM host
Base Memory4096 MB6144 MB8192 MB
Number of CPUs22–44
Disk Size40 GB
Use EFIUnchecked
Rules:
  • Keep sliders in the green area — never assign all your RAM or all CPU cores
  • Windows still needs resources to run
  • Setting too little RAM (e.g. 2 GB) will cause the installer to freeze

Click Next → review summary → Finish.

The virtual disk is a file stored on your Windows drive. It is not a new physical disk.

6Review VM settings

Before starting the VM, verify these settings:

  1. Select your Ubuntu VM → click Settings
  2. Check each section:

7Install Ubuntu

  1. Select the VM → click Start
  2. At the GRUB boot menu, select Try or Install Ubuntu and press Enter
Black screen or garbled graphics? Restart and choose Ubuntu (safe graphics) instead.
  1. Work through the Ubuntu installer:
    ScreenWhat to choose
    LanguageEnglish
    AccessibilityDefault (skip unless needed)
    KeyboardEnglish (US) or your layout
    NetworkConnect (NAT provides internet automatically)
    TypeInstall Ubuntu (not "Try")
    Installation typeInteractive Installation
    ApplicationsDefault Selection

Third-party software — check both boxes:

  • ☑ Graphics & Wi-Fi drivers — installs NVIDIA and similar proprietary drivers. In a VM this has little effect, but it's harmless.
  • ☑ Additional media formats — installs MP3, MP4, MOV codecs. Useful if you play media.
These require internet. If offline, leave unchecked — installation can continue without them.
  1. On the disk selection screen, choose "Erase disk and install Ubuntu"
    This is safe ONLY because Ubuntu is inside a VM and sees only the virtual disk. Never choose this on a physical computer unless you're sure which disk you're erasing.
  2. On the Encryption and file system screen, select No encryption
    Encryption is unnecessary for a classroom VM and adds a passphrase prompt on every boot.
  3. Click Install

"Please remove the installation medium, then press ENTER" — this is normal.
Try: Devices → Optical Drives → Remove Disk from Virtual Drive, then press Enter.
If the option is greyed out, VirtualBox already auto-ejected it — just press Enter.
Installer freeze on "Copying files"? Your VM probably has too little RAM or CPU. Power off the VM, go to Settings → System → Processor, increase to at least 4096 MB RAM / 2 CPUs. Start again — it will resume.

9First login and updates

  1. Wait for the Ubuntu login screen
  2. Select your account → enter your password
  3. Skip any online-account setup (unless instructor requires it)
  4. Open Terminal: Ctrl + Alt + T
  5. Run these one at a time: 
    sudo apt update
sudo apt upgrade -y
sudo snap refresh
When you type a password after sudo, no dots or stars appear. This is normal. Type carefully and press Enter.
  1. Restart: 
    sudo reboot

10Install Guest Additions

Guest Additions improves screen resolution, mouse integration, shared clipboard, and drag-and-drop. Two methods:

Method 1 — Graphical (try first)

  1. Start Ubuntu and log in
  2. VirtualBox menu: Devices → Insert Guest Additions CD Image
  3. If prompted, click Run → enter password → wait → restart
"bzip2 not found" error? Open a terminal and run:
sudo apt install bzip2 tar -y
Then re-insert the CD and run it again.

Method 2 — Terminal fallback

  1. Open Terminal
  2. Install dependencies: 
    sudo apt install build-essential dkms linux-headers-$(uname -r) -y
  3. Devices → Insert Guest Additions CD Image
  4. Open the Files app → open the mounted VBox_GAs disk
  5. Right-click inside → Open in Terminal
  6. Run: 
    sudo sh ./VBoxLinuxAdditions.run
  7. Restart: 
    sudo reboot

After restart — enable the features

  1. View → Auto-resize Guest Display — VM resizes with the window
  2. Devices → Shared Clipboard → Bidirectional — copy/paste between host and guest
  3. Devices → Drag and Drop → Bidirectional — drag files between host and guest

11Verify the installation

Open Terminal and run these commands:

CommandWhat it checks
whoamiLogged-in user
hostnamectlComputer name and OS info
ip aIP address
lsblkVirtual disk and partitions
df -hDisk usage
free -hRAM usage
ping -c 4 8.8.8.8Internet by IP
ping -c 4 ubuntu.comDNS resolution
Expected results: Ubuntu starts without errors, you can log in, an IP is displayed, the virtual disk is detected, ping gets replies, updates finished without errors.
To stop a ping that won't end, press Ctrl + C.

12Create a clean snapshot

A snapshot is a restore point. Create one now so you can always return to this clean state.

  1. Shut down Ubuntu properly: Power menu → Power Off → Power Off
  2. In VirtualBox Manager: select your VM → Snapshots → Take
  3. Name: Clean-Install-Updated
  4. Description: Ubuntu installed, updated, tested, and Guest Additions installed.
  5. Click OK
Do not delete this snapshot unless instructed.

13Proper shutdown

  1. Inside Ubuntu: click the Power menu
  2. Click Power Off
  3. Confirm Power Off
  4. Wait until the VM window closes
Avoid clicking "Power Off the Machine" in VirtualBox unless Ubuntu is frozen. It's like yanking the power cord — it can corrupt files.

14Troubleshooting

Problem: Ubuntu (64-bit) option is missing or VM won't start

Cause: Hardware virtualization is disabled.

Fix: Check Task Manager → Performance → CPU → Virtualization. If disabled, inform your instructor.

Problem: "No bootable medium found"

Cause: The Ubuntu ISO is not attached.

Fix: Shut down → Settings → Storage → select the optical drive → attach the Ubuntu ISO → start again.

Problem: Ubuntu installer starts again after installation

Cause: The ISO is still attached.

Fix: Remove the ISO from the virtual optical drive and restart.

Problem: No internet connection

Fix: Shut down → Settings → Network → enable Adapter 1 → set "Attached to" as NAT → start Ubuntu → run ip a and ping -c 4 8.8.8.8.

Problem: VM is very slow or freezes during install

Cause: Too little RAM or CPU.

Fix: Close unnecessary Windows apps. Increase VM to at least 4096 MB RAM / 2 CPUs. Keep sliders in the green zone.

Problem: Black screen or display glitch

Fix: Shut down → Settings → Display → set Graphics Controller to VMSVGA, Video Memory to 128 MB, disable 3D Acceleration → start again. If still broken, boot with Ubuntu (safe graphics) from the GRUB menu.

Problem: Mouse/keyboard is stuck inside the VM

Fix: Press the Host Key to release. Default on Windows: Right Ctrl.

Problem: Guest Additions — "bzip2 not found"

Fix: Run sudo apt install bzip2 tar -y then retry, or use the terminal method instead.

Problem: Forgot Ubuntu password

Fix: Do not guess repeatedly. Inform your instructor for the approved recovery procedure.
