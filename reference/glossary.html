Glossary

Key terms for SA101 VirtualBox + Ubuntu installation

Virtual Machine (VM)
A software emulation of a physical computer that runs an operating system inside your host OS. The VM sees virtual hardware (CPU, RAM, disk, network) that VirtualBox creates for it.
Host OS
The physical computer's operating system — in this case, Windows 10 or 11. VirtualBox runs on top of it.
Guest OS
The operating system running inside the VM — in this case, Ubuntu Desktop.
Hypervisor
Software that creates and runs VMs. VirtualBox is a Type 2 hypervisor (runs on top of an existing OS).
VirtualBox Guest Additions
A set of drivers and software installed inside the guest OS that improves integration: better screen resolution, mouse integration, shared clipboard, drag-and-drop, and automatic window resizing.
ISO file
A disk image file (extension .iso) containing the contents of an optical disc. The Ubuntu installer is distributed as an ISO. It is "mounted" as a virtual DVD drive inside the VM.
NAT (Network Address Translation)
Default VirtualBox network mode. The VM shares the host's IP address. The VM can browse the internet and access the host, but other devices on the physical network cannot directly reach the VM.
Dynamically allocated disk
A virtual disk file that starts small and grows as data is written, up to a configured maximum size. Saves disk space compared to a fixed-size disk that reserves all space immediately.
Snapshot
A saved state of a VM at a point in time. You can revert to it later. Useful as a restore point before making changes.
Unattended Installation
An automated installation method where VirtualBox pre-fills the Ubuntu installer options (username, password, etc.) and skips the interactive screens. SA101 requires this to be skipped so students see the manual steps.
VMSVGA
A VirtualBox graphics controller that provides better performance for Linux guests. Recommended over the default VBoxSVGA for Ubuntu.
Virtualization (VT-x / AMD-V)
Hardware features in modern CPUs that allow efficient VM execution. Must be enabled in BIOS/UEFI for 64-bit VMs to work.
sudo
A Linux command that lets you run commands with superuser (administrator) privileges. Pronounced "sue-doo" or "sue-doh."
apt
Ubuntu's package manager. apt update refreshes the list of available packages; apt upgrade installs newer versions.
snap
Ubuntu's universal package format. snap refresh updates all installed snap packages.
Host Key
The keyboard key that releases the mouse/keyboard from the VM back to the host. Default on Windows: Right Ctrl.
lsblk
A Linux command that lists block devices (disks and partitions). Used to verify the virtual disk is detected.
ping
A network utility that tests whether a remote host is reachable. Sends ICMP echo requests and waits for replies.
